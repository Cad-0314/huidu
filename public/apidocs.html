Introduction

Welcome to the VSPAY Payment Gateway API. This documentation provides a comprehensive guide to integrating our secure payment solutions for deposits (Pay-In) and withdrawals (Pay-Out).

Version: 2.1
Base URL: https://vspay.vip

All requests must be made over HTTPS with Content-Type: application/json.

Authentication

We strictly use Header-Based Authentication using HMAC-MD5 signatures. Every request must be authenticated.

Required Headers

Header Example Value Description
x-merchant-id 550e8400-e29b... Your unique Merchant UUID.
x-signature A1B2C3D4E5... The MD5 hash of your request based on sorted parameters.

Generating the Signature

Formula

1. Sort all body parameters alphabetically by key (ASCII).

2. Remove empty values and the sign field (if present).

3. Join with & in key=value format (e.g. amount=100&mId=TEST).

4. Append &secret=YOUR_MERCHANT_SECRET_KEY.

5. Hash using MD5 and convert to uppercase.

Node.js Example

const crypto = require('crypto');

function generateSignature(params, secretKey) {
    // 1. Filter empty values and 'sign' key
    const filteredParams = {};
    Object.keys(params).forEach(key => {
        if (key !== 'sign' && params[key] !== '' && params[key] != null) {
            filteredParams[key] = params[key];
        }
    });

    // 2. Sort keys
    const sortedKeys = Object.keys(filteredParams).sort();

    // 3. Build query string
    const queryString = sortedKeys.map(key => `${key}=${filteredParams[key]}`).join('&');

    // 4. Append secret
    const stringToSign = `${queryString}&secret=${secretKey}`;

    // 5. MD5 Hash (Uppercase)
    return crypto.createHash('md5').update(stringToSign).digest('hex').toUpperCase();
}

PHP Example

<?php
function generateSignature($params, $secretKey) {
    // 1. Remove empty values and 'sign'
    $filtered = array_filter($params, function($v) { return $v !== '' && $v !== null; });
    if(isset($filtered['sign'])) unset($filtered['sign']);

    // 2. Sort by key
    ksort($filtered);

    // 3. Build query string (key=value&...)
    // Note: http_build_query encodes values, we need raw parameter string? 
    // Usually Silkpay expects raw joining. manually join to avoid URL encoding issues if spec says so.
    // Spec says: "key=value" joined by "&".
    $queryParts = [];
    foreach ($filtered as $key => $value) {
        $queryParts[] = $key . "=" . $value;
    }
    $queryString = implode('&', $queryParts);

    // 4. Append secret
    $stringToSign = $queryString . "&secret=" . $secretKey;

    // 5. MD5 Hash
    return strtoupper(md5($stringToSign));
}

// Sending Request (cURL)
$merchantId = "YOUR_MERCHANT_UUID";
$secretKey = "YOUR_MERCHANT_KEY";
$url = "https://vspay.vip/api/payin/create";

$body = [
    "orderId" => "ORD-" . time(),
    "orderAmount" => 100,
    "callbackUrl" => "https://your-domain.com/callback"
];

$signature = generateSignature($body, $secretKey);

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_POST, 1);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($body));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'x-merchant-id: ' . $merchantId,
    'x-signature: ' . $signature
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$response = curl_exec($ch);
curl_close($ch);

print_r($response);
?>

Create Pay-In Order

Generate a payment link to collect funds from a user.

POST /api/payin/create

Parameter Type Description
orderId String Required Unique Order ID (Max 64 chars).
orderAmount Number Required Amount to collect (Min 100).
callbackUrl String Required Your HTTPS URL to receive webhook notifications.
skipUrl String Optional Redirect URL for the user after payment.
param String Optional Custom data to return in callback.

{

    "callbackUrl": "https://mysite.com/webhook",

We send a POST request to your callbackUrl when a transaction reaches a final state.

Notification Structure

    "id": "uuid-v4-string",     // Gateway internal ID

    "param": "custom-data",     // Your custom parameter
    "sign": "a1b2c3d4..."       // MD5 Signature
}

Verifying Callback Signature

You must verify the signature to ensure the request is from us.

Formula

1. Sort all received parameters alphabetically by key (ASCII).

2. Remove empty values and the sign field.

3. Join with & in key=value format.

4. Append &secret=YOUR_MERCHANT_SECRET.

5. Hash using MD5 and convert to uppercase.

Node.js Verification Example

function verifyCallbackSign(params, secret) {
    const receivedSign = params.sign;
    const sortedKeys = Object.keys(params).filter(k => k !== 'sign' && params[k]).sort();
    const queryString = sortedKeys.map(k => `${k}=${params[k]}`).join('&');
    const calculatedSign = crypto.createHash('md5')
                                .update(`${queryString}&secret=${secret}`)
                                .digest('hex').toUpperCase();
    return receivedSign === calculatedSign;
}
